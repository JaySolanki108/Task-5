# Task 5: Capture and Analyze Network Traffic Using Wireshark

## Objective
To capture live network traffic and analyze basic networking protocols using Wireshark as part of a cybersecurity internship task.

## Tool & Environment
- Tool: Wireshark (version 4.2.2)
- Operating System: Ubuntu Linux

## Task Description
In this task, live network packets were captured on the active network interface using Wireshark. Traffic was generated by browsing websites and using the ping command. The captured packets were then filtered and analyzed to identify different network protocols.

## Steps Performed
1. Installed and verified Wireshark on Ubuntu.
2. Selected the active network interface (wlan0).
3. Started live packet capture.
4. Generated network traffic using web browsing and ping.
5. Applied protocol filters in Wireshark.
6. Identified multiple network protocols.
7. Saved the capture file for analysis.

## Protocols Identified
- DNS (Domain Name System)
- TCP (Transmission Control Protocol)
- HTTP (Encrypted Web Traffic)

## Files in This Repository
- `task5_network_capture.pcap` / `task5_network_capture.pcapng` – Packet capture file
- `protocol_report.txt` – Short report on identified protocols
- `screenshots/` – Screenshots of installation, capture process, and protocol filtering
- `README.md` – Task documentation

## Outcome
This task provided hands-on experience in capturing and analyzing network traffic, improving understanding of packet structure and common networking protocols using Wireshark.
